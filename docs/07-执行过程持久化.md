# 执行过程持久化

## 1. 概述

持久化是 AI Agent 系统可靠性的关键保障，需要实现：
- 完整记录所有执行细节
- 支持断点恢复
- 支持人工中断和干预
- 提供审计追溯能力
- **诊断数据存储**：将大量日志/指标查询结果存入数据库，Agent 通过工具按需检索，避免撑爆上下文

```csharp
    {
        return await _context.Sessions
            .Include(s => s.Messages)
            .Include(s => s.AgentRuns)
```

## 3. 消息持久化

### 3.1 消息持久化实体

> **注意**：Framework 层的 `Message` 使用 `List<MessagePart>` 多部件结构（包含 TextPart、ToolCallPart、ToolResultPart 等多态子类）。
> 持久化层将 `Parts` 以 JSONB 整体存储，而非拆分为独立表。

```csharp
/// <summary>
/// 消息持久化实体 - 对应数据库 messages 表
/// 与 Framework 层 Message 的映射关系：
///   - Parts (JSONB) ← Message.Parts (List<MessagePart>)
///   - Metadata (JSONB) ← Message.Metadata (MessageMetadata)
/// </summary>
public class MessageEntity
{
    public Guid Id { get; set; }
    public Guid SessionId { get; set; }
    
    public string Role { get; set; }  // System, User, Assistant, Tool
    
    /// <summary>
    /// 消息内容部分，以 JSONB 存储
    /// 包含 TextPart / ToolCallPart / ToolResultPart / ImagePart / ErrorPart
    /// </summary>
    public JsonDocument Parts { get; set; }
    
    /// <summary>
    /// 消息元数据，以 JSONB 存储
    /// 包含 AgentId, EstimatedTokens, Priority, IsDeletable, Tags
    /// </summary>
    public JsonDocument Metadata { get; set; }
    
    public int EstimatedTokens { get; set; }
    
    // 来源
    public string AgentId { get; set; }
    
    public DateTime CreatedAt { get; set; }
    
    // 导航属性

}
```

### 3.2 Framework Message 与 MessageEntity 的转换

```csharp
public static class MessageEntityMapper
{
    public static MessageEntity ToEntity(Message message, Guid sessionId)
    {
        return new MessageEntity
        {
            Id = message.Id,
            SessionId = sessionId,
            Role = message.Role.ToString(),
            Parts = JsonSerializer.SerializeToDocument(message.Parts, JsonOptions),
            Metadata = JsonSerializer.SerializeToDocument(message.Metadata, JsonOptions),
            EstimatedTokens = message.Metadata.EstimatedTokens,
            AgentId = message.Metadata.AgentId,
            CreatedAt = message.CreatedAt
        };
    }
    
    public static Message ToFrameworkMessage(MessageEntity entity)
    {
        return new Message
        {
            Id = entity.Id,
            Role = Enum.Parse<MessageRole>(entity.Role),
            Parts = JsonSerializer.Deserialize<List<MessagePart>>(entity.Parts, JsonOptions),
            Metadata = JsonSerializer.Deserialize<MessageMetadata>(entity.Metadata, JsonOptions),
            CreatedAt = entity.CreatedAt
        };
    }
    
    private static readonly JsonSerializerOptions JsonOptions = new()
    {
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
        // MessagePart 多态序列化需要 TypeDiscriminator
        // 通过 System.Text.Json 的 [JsonDerivedType] 或自定义 Converter 实现
    };
}
```

### 5.3 检查点机制

检查点 **不存储** 完整的 ContextSnapshot（可能很大），而是存储消息 ID 引用列表 + 业务状态：

```csharp
public class Checkpoint
{
    public Guid Id { get; set; }
    public Guid SessionId { get; set; }
    
    public string CheckpointName { get; set; }
    public int StepNumber { get; set; }
    
    /// <summary>
    /// System Message 文本（恢复时重建 System 消息）
    /// </summary>
    public string SystemMessage { get; set; }
    
    /// <summary>
    /// 消息 ID 列表（引用 messages 表中已持久化的消息）
    /// 恢复时从 messages 表加载这些消息重建上下文
    /// </summary>
    public List<Guid> MessageIds { get; set; }
    
    /// <summary>
    /// 会话状态（业务层扩展数据）
    /// </summary>
    public JsonDocument SessionState { get; set; }
    
    /// <summary>
    /// Agent 状态（当前执行状态、迭代次数等）
    /// </summary>
    public JsonDocument AgentState { get; set; }
    
    public DateTime CreatedAt { get; set; }
}

public interface ICheckpointService
{
    Task<Checkpoint> CreateCheckpointAsync(
        Session session, 
        IContextManager contextManager,
        string name);
    Task<Checkpoint> GetLatestCheckpointAsync(Guid sessionId);
    Task<(Session, IContextManager)> RestoreFromCheckpointAsync(Guid checkpointId);
}
```

恢复流程：
1. 从 `checkpoints` 表加载 Checkpoint
2. 从 `messages` 表按 `MessageIds` 加载消息
3. 重建 `DefaultContextManager`（SetSystemMessage + AddMessages）
4. 恢复 Session 业务状态

### 5.4 基于 ContextSource 的恢复服务

使用 `ContextSource.Restored` 类型明确表达恢复意图：

```csharp
public interface ISessionRecoveryService
{
    Task<AgentResult> ResumeSessionAsync(Guid sessionId, string? continueInput = null);
    Task<bool> CanResumeAsync(Guid sessionId);
}

public class SessionRecoveryService : ISessionRecoveryService
{
    private readonly ISessionRepository _sessionRepository;
    private readonly ICheckpointService _checkpointService;
    private readonly IMessageRepository _messageRepository;
    private readonly IAgent _agent;
    private readonly ITokenEstimator _tokenEstimator;
    
    public async Task<bool> CanResumeAsync(Guid sessionId)
    {
        var session = await _sessionRepository.GetAsync(sessionId);
        return session?.Status == SessionStatus.Interrupted 
            || session?.Status == SessionStatus.WaitingApproval;
    }
    
    public async Task<AgentResult> ResumeSessionAsync(Guid sessionId, string? continueInput = null)
    {
        var session = await _sessionRepository.GetAsync(sessionId);
        
        if (!await CanResumeAsync(sessionId))
        {
            throw new InvalidOperationException(
                $"Session {sessionId} cannot be resumed from status {session.Status}");
        }
        
        // 获取最新检查点
        var checkpoint = await _checkpointService.GetLatestCheckpointAsync(sessionId);
        if (checkpoint == null)
        {
            throw new InvalidOperationException($"No checkpoint found for session {sessionId}");
        }
        
        // 从检查点恢复上下文
        var (_, contextManager) = await _checkpointService.RestoreFromCheckpointAsync(checkpoint.Id);
        
        // 添加继续输入
        if (!string.IsNullOrEmpty(continueInput))
        {
            contextManager.AddUserMessage(continueInput);
        }
        
        // 更新会话状态
        session.Status = SessionStatus.Running;
        await _sessionRepository.UpdateAsync(session);
        
        // 继续执行
        return await _agent.ExecuteAsync(contextManager);
    }
}
```

### 5.6 多 Agent 协作中的快照传递

在多 Agent 协作场景中，父 Agent 可以通过快照将上下文传递给子 Agent：

```csharp
public class CoordinatorAgent
{
    public async Task<AgentResult> DelegateToSubAgentAsync(
        IContextManager parentContext,
        IAgent subAgent,
        string task)
    {
        // 导出父上下文快照
        var parentSnapshot = parentContext.ExportSnapshot(
            parentContext.SessionId,
            new Dictionary<string, object>
            {
                [SnapshotMetadataKeys.AgentId] = Id,
                [SnapshotMetadataKeys.Reason] = "Delegation to sub-agent"
            });
        
        // 创建子 Agent 的上下文（仅传递摘要，节省 Token）
        var subContext = DefaultContextManager.StartNew(
            task,
            $"上下文摘要：{parentSnapshot.Summary}",
            _tokenEstimator);
        
        return await subAgent.ExecuteAsync(subContext);
    }
}
```

### 6.2 干预服务

```csharp
public interface IInterventionService
{
    Task InterruptSessionAsync(Guid sessionId, string reason, string userId);
    Task CancelSessionAsync(Guid sessionId, string reason, string userId);
    Task ProvideInputAsync(Guid sessionId, JsonDocument input, string userId);
    Task OverrideDecisionAsync(Guid sessionId, JsonDocument decision, string userId);
}

public class InterventionService : IInterventionService
{
    public async Task InterruptSessionAsync(Guid sessionId, string reason, string userId)
    {
        var session = await _sessionRepository.GetAsync(sessionId);
        
        if (session.Status != SessionStatus.Running)
        {
            throw new InvalidOperationException("Can only interrupt running sessions");
        }
        
        // 创建检查点（需要获取当前的 IContextManager）
        await _checkpointService.CreateCheckpointAsync(session, _contextManager, "interrupt");
        
        // 更新状态
        session.Status = SessionStatus.Interrupted;
        await _sessionRepository.UpdateAsync(session);
        
        // 记录干预
        await _interventionRepository.AddAsync(new Intervention
        {
            SessionId = sessionId,
            Type = InterventionType.Interrupt,
            Reason = reason,
            IntervenedBy = userId,
            IntervenedAt = DateTime.UtcNow
        });
        
        // 发布事件
        await _eventBus.PublishAsync(new SessionInterruptedEvent(sessionId, reason, userId));
    }
}
```

## 9. 诊断数据存储

### 9.1 设计动机

SRE Agent 在诊断过程中需要查询大量的日志（CloudWatch Logs）和指标（Prometheus Metrics）。
如果将所有查询结果直接放入 Agent 上下文，会导致：

1. **上下文溢出**：几千条日志可能消耗几万 Token，超出模型窗口限制
2. **无法精准检索**：Agent 只能看到上下文中的内容，无法按条件筛选
3. **上下文剪枝丢失数据**：当 Token 超限时，日志可能被剪枝策略删除

解决方案是将查询结果存入 `diagnostic_data` 表，Agent 只收到摘要，然后通过专门的查询工具按需检索。

### 9.2 数据流

```
┌──────────────┐     查询请求      ┌────────────────┐     原始结果       ┌───────────┐
│ Agent 调用    │ ──────────────► │  CloudWatch    │ ──────────────►  │ 结果大小   │
│ CloudWatch   │                  │  Service       │                  │ 检查      │
│ 查询工具      │                  └────────────────┘                  └─────┬─────┘
└──────────────┘                                                           │
                                                                     ┌─────┴─────┐
                                                               小于阈值│           │大于阈值
                                                                     ▼           ▼
                                                              ┌──────────┐  ┌──────────────┐
                                                              │ 直接返回  │  │ 存入 DB      │
                                                              │ 到上下文  │  │ 返回摘要     │
                                                              └──────────┘  └──────┬───────┘
                                                                                   │
                                                                                   ▼
                                                              ┌─────────────────────────────┐
                                                              │ Agent 使用查询工具按需检索     │
                                                              │ • search_diagnostic_data     │
                                                              │ • query_diagnostic_data      │
                                                              │ • get_diagnostic_summary     │
                                                              └─────────────────────────────┘
```

### 9.7 工具改造示例（CloudWatch）

```csharp
// CloudWatch 工具执行后的结果处理
public class CloudWatchSimpleQueryTool : ToolBase<CloudWatchSimpleQueryParams>
{
    private readonly IDiagnosticDataService _diagnosticDataService;
    private const int DirectReturnThreshold = 20; // 超过 20 条日志则存 DB
    
    protected override async Task<ToolResult> ExecuteAsync(
        CloudWatchSimpleQueryParams parameters,
        ToolExecutionContext context,
        CancellationToken cancellationToken)
    {
        var result = await _cloudWatchService.SimpleQueryAsync(/* ... */);
        
        if (result.Events.Count <= DirectReturnThreshold)
        {
            // 结果较少，直接返回到上下文
            return ToolResult.Success(FormatEvents(result.Events));
        }
        
        // 结果较多，存入诊断数据表
        var records = result.Events.Select(e => new DiagnosticData
        {
            SessionId = context.SessionId,
            SourceType = "cloudwatch_logs",
            SourceName = parameters.LogGroupName,
            LogTimestamp = e.Timestamp,
            Content = e.Message,
            Severity = DetectSeverity(e.Message),
            ExpiresAt = DateTime.UtcNow.AddDays(7)
        });
        
        var count = await _diagnosticDataService.StoreBatchAsync(context.SessionId, records);
        var summary = await _diagnosticDataService.GetSummaryAsync(context.SessionId, "cloudwatch_logs");
        
        // 返回摘要到上下文
        return ToolResult.Success(
            $"获取了 {count} 条日志，已存入诊断数据库。\n" +
            $"按级别统计：{FormatSeveritySummary(summary.BySeverity)}\n" +
            $"时间范围：{summary.EarliestTimestamp} ~ {summary.LatestTimestamp}\n" +
            $"使用 search_diagnostic_data 或 query_diagnostic_data 工具进行按需查询。");
    }
}
```

## 10. 数据保留与清理策略

### 10.1 数据保留策略

| 数据类型 | 保留策略 | 说明 |
|---------|---------|------|
| Session | 永久保留 | 核心业务记录 |
| Message | 永久保留 | 审计追溯需要 |
| AgentRun / ToolInvocation | 永久保留 | 审计追溯需要 |
| Checkpoint | 保留最近 5 个/Session | 旧的自动删除 |
| DiagnosticData | Session 完成后 + 7 天 TTL | 通过 `expires_at` 字段控制 |
| AuditLog | 永久保留 | 合规要求 |
| Intervention | 永久保留 | 审计追溯需要 |

### 10.2 TTL 清理服务

```csharp
public class DataCleanupService : BackgroundService
{
    private readonly IServiceScopeFactory _scopeFactory;
    private readonly ILogger<DataCleanupService> _logger;
    private readonly TimeSpan _interval = TimeSpan.FromHours(1);
    
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            try
            {
                await CleanupExpiredDataAsync(stoppingToken);
                await CleanupOldCheckpointsAsync(stoppingToken);
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "Data cleanup failed");
            }
            
            await Task.Delay(_interval, stoppingToken);
        }
    }
    
    private async Task CleanupExpiredDataAsync(CancellationToken ct)
    {
        using var scope = _scopeFactory.CreateScope();
        var context = scope.ServiceProvider.GetRequiredService<AppDbContext>();
        
        var deleted = await context.DiagnosticData
            .Where(d => d.ExpiresAt < DateTime.UtcNow)
            .ExecuteDeleteAsync(ct);
        
        if (deleted > 0)
        {
            _logger.LogInformation("Cleaned up {Count} expired diagnostic data records", deleted);
        }
    }
    
    private async Task CleanupOldCheckpointsAsync(CancellationToken ct)
    {
        using var scope = _scopeFactory.CreateScope();
        var context = scope.ServiceProvider.GetRequiredService<AppDbContext>();
        
        const int maxCheckpointsPerSession = 5;
        
        // 找出每个 Session 需要清理的旧 Checkpoint
        var sessionsWithOldCheckpoints = await context.Checkpoints
            .GroupBy(c => c.SessionId)
            .Where(g => g.Count() > maxCheckpointsPerSession)
            .Select(g => g.Key)
            .ToListAsync(ct);
        
        foreach (var sessionId in sessionsWithOldCheckpoints)
        {
            var toDelete = await context.Checkpoints
                .Where(c => c.SessionId == sessionId)
                .OrderByDescending(c => c.CreatedAt)
                .Skip(maxCheckpointsPerSession)
                .ToListAsync(ct);
            
            context.Checkpoints.RemoveRange(toDelete);
        }
        
        await context.SaveChangesAsync(ct);
    }
}
```
